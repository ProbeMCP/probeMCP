# Code of Conduct

ProbeMCP is an open-source project for embedded debugging. The project should
be useful to individuals, vendors, startups, and teams working with real
hardware.

## Expected Behavior

- Be respectful and direct.
- Focus technical discussions on evidence, reproducibility, and safety.
- Assume good intent, but do not ignore unsafe behavior.
- Document hardware risks clearly.
- Welcome new contributors while maintaining engineering quality.

## Unacceptable Behavior

- Harassment, threats, or personal attacks.
- Publishing private security details before maintainers can respond.
- Encouraging unsafe target control without clear warnings.
- Repeatedly ignoring maintainer guidance on safety-critical changes.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues; reject
contributions; or block participants when behavior harms the project or its
users.
